= AsciiDoc Notes -- Live Preview Editor for Joplin

A full-featured AsciiDoc editor plugin for https://joplinapp.org/[Joplin] with real-time live preview, a ribbon-style toolbar, and rich block editing. Write AsciiDoc notes with the same visual feedback you'd expect from a modern word processor -- rendered headings, images, tables, code blocks, diagrams, and more appear inline as you type.

Requires *Joplin 3.1+* (desktop only).

== Features at a Glance

* *Live preview* -- AsciiDoc renders inline as you type, powered by CodeMirror 6 and Asciidoctor.js
* *Ribbon toolbar* -- four tabbed panels (Text, Insert, Advanced, Editor) with one-click access to every AsciiDoc construct
* *Block editing* -- click any rendered table, code block, image, diagram, or blockquote to edit it in a focused modal or inline
* *Markdown conversion* -- convert existing Markdown notes to AsciiDoc with a single command
* *Wiki-links & cross-references* -- `+<<noteId,Display Text>>+` links between notes with autocomplete
* *Template system* -- save and reuse note templates
* *Spellcheck* -- built-in spell checker with personal dictionary support
* *Mermaid diagrams* -- 24 diagram types rendered inline with live preview editing
* *Table of Contents* -- automatic TOC generation from document headings
* *Bibliography* -- citation management with 7 format presets (APA, MLA, Chicago, IEEE, Vancouver, Harvard)
* *Favorite Copies* -- session clipboard history with Cmd+Shift+C / Cmd+Shift+V
* *Text case transforms* -- cycle through UPPER, Title Case, lower, snake_case
* *Smart quotes* -- automatic curly quote insertion with prime/double-prime support
* *Dark & light themes* -- auto-detected from Joplin's settings

'''

== Getting Started

=== Installation

. Build the plugin:
+
[source,bash]
----
npm install
npm run build
----
. The build script automatically copies the `.jpl` file to Joplin's plugin directory and clears the cache.
. *Restart Joplin* (Cmd+Q / Ctrl+Q, then reopen) -- plugins only load on startup.

=== Creating an AsciiDoc Note

* *Tools > New AsciiDoc Note* -- creates a blank AsciiDoc note
* *Tools > Convert to AsciiDoc Note* -- converts the current Markdown note in place
* *Tools > Convert to AsciiDoc Note (new file)* -- creates an AsciiDoc copy, keeping the original
* *Right-click a note > Create AsciiDoc Copy* -- convert from the note list
* *Right-click a note > Replace with AsciiDoc File* -- convert in place from the note list
* *Note toolbar button: "Make AsciiDoc"* -- one-click convert for the current note

=== Converting Entire Notebooks

* *Right-click a notebook > Create AsciiDoc Copy of Notebook* -- recursively convert all notes in the notebook to AsciiDoc copies
* *Right-click a notebook > Replace with AsciiDoc Notebook* -- recursively convert all notes in the notebook in place

You can also enable *"Create new notes as AsciiDoc"* in Preferences > AsciiDoc to auto-convert every new note.

== Live Preview

The editor renders AsciiDoc constructs inline as you type. Move your cursor to a line to see and edit the raw markup; move away and it renders as preview.

=== Supported Constructs

*Headings* -- `+= Title+` through `+===== Level 5+`, rendered with decreasing font sizes and heading colors.

*Text formatting* -- Bold, italic, monospace, strikethrough, superscript, subscript, underline, overline, highlight, and color/size roles all render inline.

*Lists* -- Bullet (`+*+`), numbered (`+.+`), checklists (`+* [x]+`), and definition lists (`+term::+`) with multi-level nesting.

*Code blocks* -- Fenced source blocks with syntax highlighting and a language label. Click to edit in a modal with a language selector.

*Tables* -- Full rendered table with header row/column support, column alignment, vertical alignment, striping patterns, frame and grid options. Click to edit with add/remove row/column controls and header toggles.

*Images* -- Web URLs and Joplin local resources (`+:/resourceId+`). Supports alt text, title, caption, scale (10-200%), alignment (center/left/right), and caption position (below/left/right). Click to open a full image editor modal.

*Admonitions* -- NOTE, TIP, WARNING, CAUTION, IMPORTANT, and QUESTION blocks with colored labels.

*Blockquotes* -- Rendered with attribution. Click to edit content and author.

*Table of Contents* -- Add `:toc:` to the document header to generate a clickable Table of Contents from section headings. Supports `:toclevels:` (1-5, default 2), `:toc-title:` (custom title), and three placement modes: `auto` (after header), `preamble` (after preamble), and `macro` (at `toc::[]` location). Click any TOC entry to jump to that heading.

*Mermaid diagrams* -- `+[mermaid]+` blocks render 24 diagram types inline: Flowchart, Sequence, Class, State, ER, Gantt, Pie, User Journey, Git Graph, Mindmap, Timeline, Quadrant, Sankey, XY Chart, Block, Packet, Kanban, Architecture, C4 (Context/Container/Component/Deployment), Requirement, and ZenUML. Click to edit in a dedicated modal with live preview. Supports width control, theme switching, and SVG caching.

*Footnotes* -- Auto-numbered superscript references [1], [2], [3]. Named footnotes (`+footnote:id[text]+`) share numbers when referenced. Click to see a floating popup with the footnote text.

*Cross-references* -- `+<<noteId,Display Text>>+` renders as a clickable link. Xrefs with section anchors (`+<<noteId#section,text>>+`) include a toggle to preview the linked section inline.

*Math / STEM* -- Inline math via `+stem:[]+`, `+latexmath:[]+`, or `+asciimath:[]+` macros renders via KaTeX directly in the preview. Display-mode block math uses `+[latexmath]+` / `+[asciimath]+` / `+[stem]+` attribute followed by `+++++` delimiters. The `:stem:` document attribute controls the default notation. AsciiMath expressions are auto-converted to LaTeX before rendering. Click a rendered block equation to edit in a modal with live preview and notation selector.

*Collapsible blocks* -- `+[%collapsible]+` blocks render with a clickable caret to expand/collapse.

*Sidebar & Example blocks* -- Rendered with appropriate borders and backgrounds.

*Horizontal rules & page breaks* -- `+'''+` renders as a divider; `+<<<+` renders as a labeled page break.

*AsciiDoc text replacements* -- `+(C)+` becomes (C), `+->+` becomes ->, `+--+` becomes an em dash, `+...+` becomes an ellipsis, and more.

== Wiki-Links & Autocomplete

Type `+<<+` to trigger note search autocomplete -- results appear as you type, showing up to 20 matching notes by title. Select a note to insert a cross-reference link.

After selecting a note, type `+#+` to drill into its sections. The autocomplete shows heading levels (h2, h3, etc.) with indentation, letting you link directly to a specific section.

'''

== Favorite Copies

A session-scoped clipboard history for frequently reused text:

. Copy text with *Cmd+Shift+C* -- the selection is copied to the system clipboard and added to the Favorite Copies list.
. Paste from the list with *Cmd+Shift+V* -- an autocomplete dropdown appears with your saved copies, showing truncated text with character counts.
. The list holds up to 20 items by default (configurable up to 100 in settings).
. The list resets when the editor session ends.

'''

== Templates

. Open any AsciiDoc note you want to reuse as a template.

== Markdown Conversion

When converting a Markdown note to AsciiDoc, the plugin automatically converts:

* *Headings*: `+# Heading+` becomes `+= Heading+` (all 6 levels)
* *Unordered lists*: `+- item+` becomes `+* item+`, with nested indentation converted to AsciiDoc nesting (`+**+`, `+***+`, etc.)
* *Links*: `+[text](url)+` becomes `+link:url[text]+`
* *Images*: `+![alt](url)+` becomes `+image::url[alt]+` (block) or `+image:url[alt]+` (inline)
* *Linked images*: `+[![alt](img)](link)+` becomes `+image::img[alt, link=link]+`
* *Inline formatting*: `+**bold**+` becomes `+*bold*+`, `+***bold italic***+` becomes `+*_text_*+`, `+~~strike~~+` becomes `+[.line-through]#text#+`
* *Code blocks*: Fenced code blocks (`++```lang++`) become AsciiDoc listing blocks (`+[source,lang]+` / `+----+`)
* *Horizontal rules*: `+---+`, `+***+`, `+___+` become `+'''+`
* *HTML cleanup*: `+<br>+` tags become newlines, common inline HTML tags are stripped
* *Escape removal*: Markdown backslash escapes (`+\*+`, `+\[+`, etc.) are cleaned up
* Content inside fenced code blocks is left untouched

== Document Attributes

Use *Tools > Edit AsciiDoc Attributes* to set document-level attributes (author, revision, custom variables). These are stored in the note's sentinel block and passed to Asciidoctor during rendering.

Supported document attributes in the editor header:

[cols="1,1,3"]
|===
| Attribute | Default | Description

| `:toc:`
| _(not set)_
| Enable Table of Contents. Values: `auto`, `preamble`, `macro`

| `:toclevels:`
| 2
| Number of heading levels shown in TOC (1-5)

| `:toc-title:`
| Table of Contents
| Custom title for the TOC box

| `:stem:`
| _(not set)_
| Enable math rendering. Values: `latexmath`, `asciimath`
|===

Custom key=value attributes can also be set via the dialog and are passed through to Asciidoctor.

== Architecture

The plugin consists of two webpack bundles:

[cols="1,1,1,3"]
|===
| Bundle | Target | Size | Contains

| `index.js`
| Node (plugin sandbox)
| ~822 KB
| Plugin registration, commands, settings, IPC handlers, Markdown-to-AsciiDoc converter, Asciidoctor.js rendering

| `panel.js`
| Web (webview)
| ~5.5 MB
| CodeMirror 6 editor, live-preview decoration engine, KaTeX math rendering, Mermaid diagram rendering (with ELK layout), nspell spellcheck, toolbar, IPC client
|===

AsciiDoc notes are identified by a `++```asciidoc-settings++` sentinel block appended to the note body. This block stores document attributes as JSON and is stripped before rendering.

Communication between the plugin sandbox and the webview editor uses Joplin's `postMessage` IPC protocol (19 message types from webview to plugin, 4 from plugin to webview) for operations like saving, rendering, resource resolution, note search, template management, spellcheck settings, and fullscreen mode.
